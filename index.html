frequently asked questions

got questions? we got answers.

[?]

what executors does this work on?

any executor that at least has file system functionality. this is basically every executor right now. also, nozomi does work on lower unc external execs like solara (havent tested on xeno but should work) so uh yeah... future features might not work on lower unc executors though.

[?]

is this detected? will i get banned?

okay lets be fr. the script itself? probably to some degree (especially on advanced anticheats), but most likely not enough to cause an in game ban unless you abuse blatant modules (speed, fly, etc) too much. please note that most desktop (windows) based executors are detected, so i'd use an alt unless you're on mac, where ud options like hydrogen exist.

[?]

how do i change the theme?

once you execute the script, open the UI and navigate to the "theme" tab. there you'll find some pre-made themes to choose from. you can also customize individual colors, adjust UI transparency, and change fonts with even more customization coming soon. all settings can be saved to your executor's workspace folder.

[?]

what games are supported?

nozomi is a universal script hub, so most core features (aimbot, esp, and movement to some degree) should work in fps/combat games. we also have game-specific modules for blade ball, gunfight arena, and 99 nights at the moment. more game specific modules are coming soon (if you got some good game suggestions dm me on discord: xqtn_ im running out of ideas)

[?]

how do i report bugs or request features?

dm me on discord: xqtn_. please tell me which executor you're using, steps to recreate the bug(s), and screenshots/videos of the bug.

[?]

does it auto-update?

yes. the loadstring pulls directly from github, so you automatically get the latest version every time you execute. the UI library is cached locally for faster load times, but it checks for updates every time a new version hash is detected. you can force the script to download the ui library again by deleting the "nozomi_cache" folder in your executor's workspace folder.

[?]

is it free? is there a key system? and if it is, why is it free?

yeah this script is completely free to use, currently i don't have any plans to directly monetize it. as of right now, there's no key system, but i might decide to add one in the future (dont worry, it'll have extremely long key lengths, so you only have to do it once every like... week or two. i am not greedy asf unlike some other devs. i just need enough to renew this domain.) as for why it's free? its because this is essentially fully a passion project. i got bored one day and i decided to start making this.

[?]

is this safe? does this log my information? and is it open source?

kinda depends on what you say "safe" is. the actual code behind nozomi is not malicious. and yes. we do collect imformation, however it's non identifiable. we only collect data such as executor, if the script loaded successfully, and basic post requests to update our execution counter. we do NOT log your ip directly, however since we use cloudflare as protection, they might log your ip, but only the region is viewable. we cannot view your ip. also, no. nozomi is closed source.
